## Folder `functions/`

This directory contains the core functional scripts of the data processing pipeline, covering the preprocessing of raw motion capture data (CSV to C3D), post-processing and conversion of MoSh++ results, and the final visualization and validation of the motion data.

### 📂 Directory Structure

```text
functions/
├── process_file.py          # Data preprocessing and format conversion
├── recover_24_smpl.py       # Skeletal topology mapping and dimensionality reduction
└── visualization_2d+3d.py   # Result rendering and visualization validation

```

---

### Script Details

#### 1. `process_file.py`

* **Description:** Responsible for the data preprocessing required before running the MoSh++ algorithm. It converts the `.csv` motion trajectories (split and generated by external software) into the standard `.c3d` format required by MoSh++.
* **Core Logic:** Reads CSV coordinate data, cleans, and filters it. For each 3D coordinate across all frames, it introduces a `residual` (confidence) label matrix: valid and normal points are marked as `1.0`, while missing or invalid values (such as NaNs or all-zero coordinates) are marked as `-1.0`. This ensures that MoSh++ can correctly identify and handle missing data.
* **Configuration:** Before running, you must modify the absolute paths for the source data (`input_path`) and the exported data (`output_path`) directly within the script.

#### 2. `recover_24_smpl.py`

* **Description:** Used for result post-processing and data dimensionality reduction after the MoSh++ `Stage II` computation is complete.
* **Core Logic:** By default, MoSh++ outputs sequences based on high-precision model parameters. This script loads the official SMPL and SMPL-X models, along with the `smplx_to_smpl` mapping matrix, to reconstruct and map these complex parameters back to **standard 24-joint SMPL 3D coordinates**. The results are saved in `.pkl` format, which significantly simplifies data loading and preprocessing for downstream deep learning models.
* **Configuration:** Please strictly verify the following paths before execution:
* The **input and output data paths**.
* The **paths for model dependency files**: `SMPL`, `SMPL-X`, and the mapping file. These are stored by default in the `2026_moshpp_for_latest_data/soma_2026_workspace/support_files` directory.



#### 3. `visualization_2d+3d.py`

* **Description:** A visualization and quality assessment tool for the reconstructed results.
* **Core Logic:** Reads the post-processed 3D coordinate data (`.pkl`) and provides a dual-view alignment validation:
1. **2D Projection Validation:** Back-projects the 3D joints and overlays them onto the original video to check spatial alignment accuracy.
2. **3D Spatial Rendering:** Reconstructs the motion trajectory in a pure 3D coordinate system to check for depth anomalies or severe joint distortions.


* **Configuration:** Before running, configure the **paths for the input data** (the `.pkl` file and the original video) as well as the **output save path** for the rendered results.
